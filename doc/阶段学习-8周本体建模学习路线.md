## 阶段一：零基础入门（第1–3周）
**目标：搞懂本体是什么，会用 Protégé 建一个简单本体**

### 第1周：概念 + 工具上手
**学习内容**
- 什么是本体：类、属性、实例、公理、推理
- RDF / RDFS / OWL 的基本区别
- Protégé 界面与基本操作

**必看资料**
- 论文/小册子：《Ontology Development 101》（Stanford）
- B站/CSDN 搜：**“Protégé OWL 入门教程”**（选一个播放量高的完整视频）

**本周实操**
1. 安装 Protégé（最新版）
2. 新建一个本体：`University`
3. 建 3 个类：`Person`、`Student`、`Teacher`
4. 设置 `Student` ⊑ `Person`，`Teacher` ⊑ `Person`
5. 建一个对象属性 `teaches`，定义域 `Teacher`，值域 `Student`
6. 创建 1 个 Teacher 实例、2 个 Student 实例，连上 `teaches`

✅ 产出：一个 `.owl` 文件 + 一张本体结构截图

---

### 第2周：属性体系 + 约束
**学习内容**
- 对象属性 vs 数据属性
- 功能属性（Functional）、逆属性（Inverse）、传递性（Transitive）
- 基数约束（Cardinality）、全称量词/存在量词

**实操任务**
- 在上周本体中增加：
  - 数据属性：`hasName`（xsd:string）、`hasAge`（xsd:int）
  - 对象属性：`enrolledIn`（学生–课程）
- 给 `Student` 加约束：  
  - 每个学生至少选修 1 门课  
  - 每门课最多有 50 个学生

✅ 产出：带约束的 University v2.owl

---

### 第3周：推理机初体验
**学习内容**
- 为什么需要推理：隐含知识自动推导
- Protégé 内置推理机：HermiT / FaCT++

**实操任务**
- 定义：
  - `UndergraduateStudent` ⊑ `Student`
  - `GraduateStudent` ⊑ `Student`
  - 约束：本科生年龄 ≤ 22
- 创建一个 25 岁的学生实例，不手动标类型
- 运行推理机，观察是否自动归类为 `GraduateStudent`

✅ 产出：推理结果截图 + 一句话总结“推理帮我省了什么手工工作”

---

## 阶段二：进阶推理与本体工程（第4–6周）
**目标：掌握本体设计方法、对齐、评估，能处理稍复杂的领域**

### 第4周：本体构建方法论
**学习内容**
- 本体工程流程：需求 → 术语收集 → 建模 → 评估 → 维护
- 常见陷阱：类 vs 实例混淆、过度细分、命名不规范

**实操任务**
- 选一个你熟悉的领域（如图书、电影、医疗、供应链）
- 写一页《本体需求说明书》：
  - 覆盖范围
  - 核心类（10 个以内）
  - 关键关系
- 用 Protégé 实现该本体的 V1 版本

---

### 第5周：顶层本体 & 复用
**学习内容**
- 常见顶层本体：SUMO、BFO、DOLCE
- 为什么要复用现有本体（减少重复、提高互操作性）

**实操任务**
- 下载 SUMO 或 BFO 的 OWL 文件
- 将你上周的领域本体与顶层本体对齐（如把 `Person` 映射到 SUMO 的 `Human`）
- 体验“站在巨人肩膀上”的感觉

---

### 第6周：SPARQL 查询 + 三元组存储
**学习内容**
- RDF 三元组模型
- SPARQL 基本语法：SELECT、WHERE、FILTER

**实操任务**
- 使用 Apache Jena Fuseki 或 GraphDB 免费版
- 导入你的本体
- 写 5 条 SPARQL 查询，例如：
  - “所有年龄大于20的学生”
  - “教授某门课的老师的姓名”

✅ 产出：SPARQL 查询脚本 + 查询结果截图

---

## 阶段三：本体 × 大模型（第7–8周）
**目标：理解本体在大模型时代的新角色：增强语义、减少幻觉**

### 第7周：本体 + LLM 基础认知
**学习内容**
- 大模型为什么“不懂语义”：token 不等于概念
- 本体如何帮助 LLM：
  - 提供结构化知识
  - 约束生成空间
  - 支持知识校验

**阅读推荐**
- 搜索关键词：**“ontology large language model”、“knowledge graph prompting”**
- 看 2～3 篇综述或博客（不必深挖公式）

**实操任务**
- 设计一个 Prompt：  
  - 先给大模型一段本体片段（类、属性、约束）
  - 再让它基于本体回答问题或生成结构化数据
- 对比“有无本体”的回答质量差异

---

### 第8周：小型综合项目（毕业作业）
**做一个“本体 + 程序 + 大模型”的小 Demo**

示例选题（任选其一）：
1. **课程知识图谱问答**
   - Protégé 建课程本体
   - Fuseki 存数据
   - Python + SPARQL 查数据
   - LLM 根据查询结果生成自然语言回答

2. **企业数据治理小本体**
   - 建模“部门–员工–项目–资产”
   - 用 OWL 约束“一个资产只能属于一个部门”
   - 用推理发现违规分配

3. **医疗/法律概念校验**
   - 用小本体定义核心概念
   - 让 LLM 生成若干陈述
   - 用推理机判断是否存在逻辑冲突

✅ 最终交付：
- 本体文件（.owl）
- 简要设计说明（1～2 页）
- 演示截图或录屏

## 给你一个小建议
本体建模**最忌“贪大求全”**。  
建议你前 4 周都只用「University / Bookstore / Movie」这种极小领域，**先把“类–属性–约束–推理”这四条线练熟**，后面再碰大模型才会真正有体感。
