# 二、University 本体升级：带推理的版本（提前体验）

下面是在你刚完成的 `university-v1.owl` 基础上，**加上 OWL 约束，让推理机“自动干活”**。

## 场景设定
我们要表达两条规则：
1. **本科生（UndergraduateStudent）年龄 ≤ 22**
2. **研究生（GraduateStudent）年龄 > 22**
3. 一个学生如果年龄 25，**不用手动标类型**，推理机应能自动判断他是研究生。

---

## Step 1：新增两个子类

1. 进入 **Classes**
2. 选中 `Student`
3. 新建子类：
   - `UndergraduateStudent`
   - `GraduateStudent`

---

## Step 2：用 Data Property Restrictions 表达“年龄约束”

### 定义 UndergraduateStudent
1. 选中 `UndergraduateStudent`
2. 右侧 **Description → Disjoint with** → 点击 **+**
3. 选择 `GraduateStudent`（两者互斥）
4. 右侧 **SubClass Of** → 点击 **+ → Class expression editor**
5. 输入：
   ```text
   hasAge some xsd:int[<= 22]
   ```
6. OK

👉 含义：  
**本科生 = 年龄不超过 22 的学生**

---

### 定义 GraduateStudent
1. 选中 `GraduateStudent`
2. SubClass Of → +
3. 输入：
   ```text
   hasAge some xsd:int[> 22]
   ```
4. OK

👉 含义：  
**研究生 = 年龄大于 22 的学生**

---

## Step 3：创建一个“未标注类型”的学生

1. 切换到 **Individuals**
2. 新建个体：`Charlie`
3. 类型先只选 `Student`
4. 添加数据属性：
   - `hasAge = 25`

⚠️ 注意：**不要手动给 Charlie 加 GraduateStudent**

---

## Step 4：启动推理机（见证魔法）

1. 顶部菜单 → **Reasoner**
2. 选择 **HermiT**
3. 点击 **Start reasoner**

观察左侧 Individuals 面板：
- `Charlie` 下方会自动出现：
  ```
  GraduateStudent
  ```

✅ **推理机根据你定义的规则，自动完成了分类！**

---

## Step 5：制造一个“矛盾”试试（非常重要）

1. 给 `Charlie` 再加一个数据属性：
   - `hasAge = 20`
2. 回到 Reasoner → **Synchronize reasoner**

👉 你会看到 **红色报错 / Inconsistent Ontology**  
因为：
- 25 > 22 → GraduateStudent
- 20 ≤ 22 → UndergraduateStudent
- 两者互斥 → 矛盾

🎯 **这正是本体的价值：机器帮你发现人类容易忽略的逻辑冲突。**
