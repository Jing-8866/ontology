## 第6周：SPARQL 查询与三元组存储（Apache Jena Fuseki）

**目标**：把 Protégé 建的本体放进服务器，用 SPARQL 查数据。

### Step 1 安装启动 Fuseki
1. 访问 **https://jena.apache.org/download/** 下载 `apache-jena-fuseki-*-distribution.zip`。
2. 解压 → 进入目录 → 双击 `fuseki-server.bat`（Win）或运行 `./fuseki-server`（Mac/Linux）。
3. 浏览器打开 **http://localhost:3030**，看到 Fuseki 管理界面。

### Step 2 导出本体为 Turtle
1. 回到 Protégé → **File → Export → Turtle (.ttl)** → 存为 `library.ttl`。

### Step 3 在 Fuseki 中建 Dataset 并上传
1. Fuseki 页面点 **Manage Datasets → Add Dataset**。
2. Dataset Name 填 `library`，类型选 **In-memory (with persistence)** → **Create**。
3. 切到 **upload data** 页 → 选 `library.ttl` → **Upload all**。

### Step 4 写第一条 SPARQL
1. 切到 **Query** 页，Dataset 选 `library`。
2. 在输入框贴入：
   ```sparql
   PREFIX lib: <http://www.example.org/library#>
   PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

   SELECT ?book ?title
   WHERE {
     ?book rdf:type lib:Book .
     ?book lib:hasTitle ?title .
   }
   ```
3. 点 **Run Query**，下方表格应出现 `AI_Book_01 | "人工智能导论"`。

### Step 5 进阶查询（带属性的过滤）
```sparql
SELECT ?book ?year
WHERE {
  ?book lib:hasYear ?year .
  FILTER (?year > 2020)
}
```

## 全程避坑复查（贯穿8周）

- **每周保存都用 Save As 带版本号**（week1-v1, week2-v1…），方便回滚。
- **推理机启动前先 Check consistency**，红报错再看 Explanation（右键红色类 → Explain），不要盲目关推理。
- **SPARQL 查不到数据先核对 Prefix**，你的 IRI 是 `example.org` 还是 `semanticweb.org`，必须和文件里一致。
- **对齐顶层本体时注意命名冲突**，导入 SUMO 后别把自己的类也命名成 `Agent`，可用 `lib:Person` 前缀隔离。
