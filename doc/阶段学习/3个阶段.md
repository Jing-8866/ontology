## 第一阶段：零基础入门（第1–3周）——Protégé 动手为主

### 第1周：概念 + 工具上手（University 本体 V1）
**目标**：安装 Protégé，建类、属性、实例，产出 `.owl` 文件。

- **Step 1 环境准备**
  1. 确认电脑已装 **Java 11+**（Protégé 5.5+ 依赖）。
  2. 到 https://protege.stanford.edu/ 下载 **Protégé Desktop**。
  3. 解压后运行 `protege.bat`（Win）或 `protege.sh`（Mac/Linux）。

- **Step 2 新建本体**
  1. 启动后点 **File → New**。
  2. 在 **Ontology IRI** 填 `http://www.example.org/university#`。
  3. 点 **OK**，进入 **Active Ontology** 页签。

- **Step 3 建类（Classes）**
  1. 左侧切到 **Entities → Classes**。
  2. 选中 `owl:Thing` → 点 **Add subclass**（⊕）→ 输 `Person` → 回车。
  3. 选中 `Person` → 建子类 `Student`、`Teacher`。
  4. 结构应为：
     ```
     owl:Thing
      └── Person
           ├── Student
           └── Teacher
     ```

- **Step 4 建对象属性（Object Properties）**
  1. 切到 **Object Properties**。
  2. 选 `owl:topObjectProperty` → **Add subproperty** → 输 `teaches`。
  3. 右侧 **Domain** 点 `+` 选 `Teacher`；**Range** 点 `+` 选 `Student`。

- **Step 5 建数据属性（Data Properties）**
  1. 切到 **Data Properties**。
  2. 建 `hasName`（Range 选 `xsd:string`）、`hasAge`（Range 选 `xsd:int`）。

- **Step 6 建实例（Individuals）**
  1. 切到 **Individuals by class**。
  2. 右击 `Teacher` → **Add individual** → 输 `Alice`，在 **Data property assertions** 加 `hasName="Alice"`、`hasAge=35`。
  3. 同理建 `Student` → `Bob`（`hasAge=20`）。
  4. 选 `Alice` → **Object property assertions** → `+` → `teaches` → 选 `Bob`。

- **Step 7 保存**
  1. **File → Save** → 存为 `university-v1.owl`（选 RDF/XML 格式）。

### 第2周：属性体系 + 约束（University V2）
**目标**：加 `enrolledIn`、基数约束、互斥类。

- **Step 1 新建类 `Course`**
  1. Classes 下 `owl:Thing` → 建 `Course`。

- **Step 2 建对象属性 `enrolledIn`**
  1. Object Properties → 建 `enrolledIn`。
  2. Domain：`Student`；Range：`Course`。

- **Step 3 加基数约束**
  1. 选中 `Student` 类。
  2. 右侧 **SubClass Of** → `+` → **Class expression editor**。
  3. 输：`enrolledIn min 1 Course`（至少选1门课）。
  4. 再 `+` → 输：`enrolledIn max 5 Course`（最多5门）。

- **Step 4 设互斥（Disjoint）**
  1. 选中 `Student` → 右侧 **Disjoint With** → `+` → 选 `Teacher`。
  2. 同理 `Teacher` disjoint with `Student`。

- **Step 5 保存为 `university-v2.owl`**

### 第3周：推理机初体验（自动分类）
**目标**：用 OWL 限制让推理机自动判类型。

- **Step 1 建子类**
  1. `Student` 下建 `UndergraduateStudent`、`GraduateStudent`，并互相 **Disjoint With**。

- **Step 2 写等价类/限制**
  1. 选 `UndergraduateStudent` → **SubClass Of** → `+` → 输：
     `hasAge some xsd:int[<= 22]`
  2. 选 `GraduateStudent` → **SubClass Of** → `+` → 输：
     `hasAge some xsd:int[> 22]`

- **Step 3 建模糊实例**
  1. 建个体 `Charlie`，类型只给 `Student`，`hasAge=25`，**不手动标 GraduateStudent**。

- **Step 4 启动推理**
  1. 菜单 **Reasoner → HermiT → Start reasoner**。
  2. 观察 `Charlie` 下是否**自动出现 `GraduateStudent`**。

- **Step 5 制造矛盾测试**
  1. 给 `Charlie` 再加 `hasAge=20` → 同步推理 → 看 **Inconsistent Ontology** 报错。

## 第二阶段：进阶推理与本体工程（第4–6周）——方法与数据层

### 第4周：本体构建方法论（七步法实操）
**参考斯坦福七步法**：
1. **确定领域和范围**：写一页《本体需求说明书》，列 Competency Questions（如“哪门课由 Alice 教？”）。
2. **考虑复用**：去 https://protege.stanford.edu/plugins.html 或 SUMO/BFO 官网下载顶层本体。
3. **枚举术语**：列出领域核心词（课程、院系、学分、学期）。
4. **定义类层次**：在 Protégé 里按自顶向下建类树。
5. **定义属性**：区分对象属性（师生、授课）和数据属性（姓名、学分）。
6. **定义约束**：用 cardinality、domain/range、disjoint 完善。
7. **创建实例**：填 10–20 个实例验证。

👉 **保姆级动作**：每做完一步，就在 Protégé 里 **Save As** 一个新版本，如 `week4-v1.owl`。

### 第5周：顶层本体对齐
- **Step 1** 下载 **SUMO** OWL 文件（IEEE SUO 官网）。
- **Step 2** 在 Protégé 里 **File → Import → Import Ontology from URL/File**，导入 SUMO。
- **Step 3** 把你自己的 `Person` 类右侧 **Equivalent To** 或 **SubClass Of** 关联到 SUMO 的 `Human`。
- **Step 4** 运行推理机，检查是否能继承 SUMO 里的通用公理。

---

### 第6周：SPARQL + 三元组存储
**目标**：用 Apache Jena Fuseki 查本体。

- **Step 1** 下载 Fuseki（https://jena.apache.org/），解压运行 `fuseki-server`。
- **Step 2** Protégé 里 **File → Export → Turtle**，导出 `university.ttl`。
- **Step 3** 浏览器打开 `http://localhost:3030`，建 dataset，上传 `university.ttl`。
- **Step 4** 在 Query 页写 SPARQL：
  ```sparql
  SELECT ?s WHERE {
    ?s a <http://www.example.org/university#Student> .
  }
  ```
- **Step 5** 依次写 5 条查询（按年龄、按课程、按老师等）。

## 第三阶段：本体 × 大模型（第7–8周）——综合项目

### 第7周：本体增强 Prompt
- **Step 1** 在 Protégé 里导出一小段本体片段（Manchester Syntax 或 Turtle）。
- **Step 2** 构造 Prompt：
  ```
  以下是大学本体的类与属性定义：
  Person ⊑ owl:Thing
  Student ⊑ Person
  teaches Domain: Teacher, Range: Student
  属性 hasAge(xsd:int)

  问题：Alice 是 Teacher，教了 Bob，Bob 年龄 20，请判断 Bob 是否是 UndergraduateStudent？
  ```
- **Step 3** 对比：先不给本体片段让 LLM 回答，再给本体片段回答，记录差异。

---

### 第8周：小型综合项目（毕业 Demo）
**以“课程知识图谱问答”为例：**
1. Protégé 建完整课程本体（大学、院系、教师、学生、课程、教室）。
2. Fuseki 导入，写 SPARQL 支持“查某老师所有课”“查某课所有学生”。
3. Python 用 `owlready2` 或 `rdflib` 读本体，调 LLM API。
4. 流程：
   - 用户问自然语言 → LLM 抽取意图 → 转 SPARQL → Fuseki 查 → 结果返给 LLM → 生成回答。
5. 截屏/录屏，整理成 **说明文档 + owl 文件 + 查询脚本**。
