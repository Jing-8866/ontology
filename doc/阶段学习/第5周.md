# 第四阶段：进阶推理与本体工程（第4–6周）

---

## 第5周：顶层本体对齐与复用

**目标**：把自建类映射到 SUMO/BFO/schema.org，体验本体互操作。

### Step 1 下载顶层本体
1. 访问 SUMO 官网或 **https://www.adampease.org/OP/** 下载 `sumo.owl`。
2. 或直接用 schema.org 的 OWL 版本（Google 搜 `schemaorg owl file` 下载）。

### Step 2 导入本体到当前工程
1. Protégé 菜单 → **File → Import → Import Ontology...**
2. 选 **From File...** → 选中下载的 `sumo.owl` → 一路 **Next** → **Finish**。
3. 左侧 Classes 里会出现大量 SUMO 的类（如 `Human`、`Artifact`）。

### Step 3 建立对齐关系
1. 选中自建类 `Person`。
2. 右侧 **SubClass Of** → `+` → **Class expression editor** → 输：
   ```
   sumo:Human
   ```
   （或直接在下拉里选导入的 SUMO Human）
3. 同理，`Book` 设为 `sumo:Artifact` 的子类。
4. **保存**，此时你的本体已“挂载”在顶层本体上。

### Step 4 推理验证对齐效果
1. **Reasoner → HermiT → Start reasoner**。
2. 新建一个 `Author` 实例 `LiSi`，推理机会自动推导它也是 `sumo:Human`，在类面板 `Human` 下能看到 `LiSi`（虚线显示）。

## 全程避坑复查（贯穿8周）

- **每周保存都用 Save As 带版本号**（week1-v1, week2-v1…），方便回滚。
- **推理机启动前先 Check consistency**，红报错再看 Explanation（右键红色类 → Explain），不要盲目关推理。
- **SPARQL 查不到数据先核对 Prefix**，你的 IRI 是 `example.org` 还是 `semanticweb.org`，必须和文件里一致。
- **对齐顶层本体时注意命名冲突**，导入 SUMO 后别把自己的类也命名成 `Agent`，可用 `lib:Person` 前缀隔离。
