# 第五阶段：本体 × 大模型（第7–8周）

## 第7周：本体增强 Prompt 工程（Step-by-Step）

**目标**：用本体片段约束 LLM，减少幻觉。

### Step 1 提取本体片段（Manchester Syntax）
1. Protégé 中选类 `Book` → 右侧 **Description**，复制文本：
   ```
   Class: Book
     SubClassOf: 
       hasAuthor some Author,
       belongsToCategory some Category,
       hasTitle exactly 1 xsd:string
   ```
2. 同理复制 `Author`、`ComputerScience` 的定义。

### Step 2 构造本体增强 Prompt
在 LLM 对话框（ChatGPT/Claude）中输入：
```
以下是图书管理本体的OWL定义（Manchester Syntax）：
---
Class: Book
  SubClassOf: hasAuthor some Author, belongsToCategory some Category
Class: Author
  SubClassOf: Person
Class: ComputerScience
  SubClassOf: Category
---
问题：如果一本书《深度学习》属于ComputerScience，它的作者张三应该是什么类型？请严格基于上述本体回答，不要引入外部知识。
```

### Step 3 对比实验
1. **不加本体**，直接问同上问题，记录 LLM 是否瞎编属性。
2. **加本体**，观察回答是否严格限制在 `Author` 类型内。
3. 把两段回答存进一个 txt，标注差异（如：有无提到 `publishedBy` 等未定义属性）。

## 全程避坑复查（贯穿8周）

- **每周保存都用 Save As 带版本号**（week1-v1, week2-v1…），方便回滚。
- **推理机启动前先 Check consistency**，红报错再看 Explanation（右键红色类 → Explain），不要盲目关推理。
- **SPARQL 查不到数据先核对 Prefix**，你的 IRI 是 `example.org` 还是 `semanticweb.org`，必须和文件里一致。
- **对齐顶层本体时注意命名冲突**，导入 SUMO 后别把自己的类也命名成 `Agent`，可用 `lib:Person` 前缀隔离。
