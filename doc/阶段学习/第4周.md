# 进阶推理与本体工程（第4–6周）

## 第4周：本体工程方法论（斯坦福七步法落地）

**目标**：从“随便建”转向“规范建模”，产出《本体需求说明书》与 V1 领域本体。

### Step 1 确定领域与范围（纸上作业）
1. 打开记事本或 Word，写下标题 **《图书管理本体需求说明书》**。
2. 列出 **Competency Questions（能力问题，即这个本体要回答什么问题）**：
   - 某本书的作者是谁？
   - 某位作者写了哪些书？
   - 哪本书属于计算机分类？
3. 确定范围：只含图书、作者、分类、出版社，不含借阅记录（避免过大）。

### Step 2 考虑复用现有本体
1. 浏览器打开 **http://schema.org**（或下载 FOAF 本体）。
2. 记下可复用的类：`schema:Book`、`schema:Person`、`schema:Organization`。
3. Protégé 中先不导入，心里有数后面要对齐。

### Step 3 枚举术语与建类
1. 启动 Protégé → **File → New** → IRI 填 `http://www.example.org/library#`。
2. **Classes** 标签 → 选 `owl:Thing` → **Add subclass**：
   - 输 `Book` → 回车
   - 选 `owl:Thing` → 建 `Person`、`Category`、`Publisher`
3. 选 `Person` → **Add subclass** → 输 `Author`。
4. 选 `Category` → 建子类 `ComputerScience`、`Literature`。
5. 右侧 **Disjoint with** → 分别把 `Book`、`Person`、`Category`、`Publisher` 设为**两两互斥**。

### Step 4 定义属性
1. **Object Properties** → 选 `owl:topObjectProperty` → **Add subproperty**：
   - `hasAuthor`（Domain: Book, Range: Author）
   - `belongsToCategory`（Domain: Book, Range: Category）
   - `publishedBy`（Domain: Book, Range: Publisher）
2. **Data Properties** → 建：
   - `hasTitle`（Range: xsd:string）
   - `hasISBN`（Range: xsd:string）
   - `hasYear`（Range: xsd:int）

### Step 5 创建实例验证
1. **Individuals** → 右击 `Book` → **Add individual** → 输 `AI_Book_01`。
2. 右侧 **Data property assertions** → `+` → `hasTitle` → `"人工智能导论"`；`hasISBN` → `"978-XXXX"`；`hasYear` → `2023`。
3. 建个体 `ZhangSan` 类型 `Author`，用 `hasAuthor` 连到 `AI_Book_01`。

### Step 6 保存版本
- **File → Save As** → `library-week4-v1.owl`（选 RDF/XML）。

## 全程避坑复查（贯穿8周）

- **每周保存都用 Save As 带版本号**（week1-v1, week2-v1…），方便回滚。
- **推理机启动前先 Check consistency**，红报错再看 Explanation（右键红色类 → Explain），不要盲目关推理。
- **SPARQL 查不到数据先核对 Prefix**，你的 IRI 是 `example.org` 还是 `semanticweb.org`，必须和文件里一致。
- **对齐顶层本体时注意命名冲突**，导入 SUMO 后别把自己的类也命名成 `Agent`，可用 `lib:Person` 前缀隔离。
