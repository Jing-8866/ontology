## 第二阶段：进阶推理与本体工程（第4–6周）——方法与数据层

### 第4周：本体构建方法论（七步法实操）
**参考斯坦福七步法**：
1. **确定领域和范围**：写一页《本体需求说明书》，列 Competency Questions（如“哪门课由 Alice 教？”）。
2. **考虑复用**：去 https://protege.stanford.edu/plugins.html 或 SUMO/BFO 官网下载顶层本体。
3. **枚举术语**：列出领域核心词（课程、院系、学分、学期）。
4. **定义类层次**：在 Protégé 里按自顶向下建类树。
5. **定义属性**：区分对象属性（师生、授课）和数据属性（姓名、学分）。
6. **定义约束**：用 cardinality、domain/range、disjoint 完善。
7. **创建实例**：填 10–20 个实例验证。

👉 **保姆级动作**：每做完一步，就在 Protégé 里 **Save As** 一个新版本，如 `week4-v1.owl`。

---

### 第5周：顶层本体对齐
- **Step 1** 下载 **SUMO** OWL 文件（IEEE SUO 官网）。
- **Step 2** 在 Protégé 里 **File → Import → Import Ontology from URL/File**，导入 SUMO。
- **Step 3** 把你自己的 `Person` 类右侧 **Equivalent To** 或 **SubClass Of** 关联到 SUMO 的 `Human`。
- **Step 4** 运行推理机，检查是否能继承 SUMO 里的通用公理。

---

### 第6周：SPARQL + 三元组存储
**目标**：用 Apache Jena Fuseki 查本体。

- **Step 1** 下载 Fuseki（https://jena.apache.org/），解压运行 `fuseki-server`。
- **Step 2** Protégé 里 **File → Export → Turtle**，导出 `university.ttl`。
- **Step 3** 浏览器打开 `http://localhost:3030`，建 dataset，上传 `university.ttl`。
- **Step 4** 在 Query 页写 SPARQL：
  ```sparql
  SELECT ?s WHERE {
    ?s a <http://www.example.org/university#Student> .
  }
  ```
- **Step 5** 依次写 5 条查询（按年龄、按课程、按老师等）。
