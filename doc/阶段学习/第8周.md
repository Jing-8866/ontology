# 第五阶段：本体 × 大模型（第7–8周）

## 第8周：综合项目——“课程知识图谱问答 Demo”

**目标**：打通“Protégé 建模 → Fuseki 存储 → SPARQL 查询 → LLM 生成”全流程。

### Step 1 完善 University 本体（终极版）
在 **第3周带推理版本** 基础上补充：
1. 加类 `Course`、`Department`。
2. 加属性 `offeredBy`（Course → Department）、`enrolledIn`（Student → Course）。
3. 给 `Course` 加基数约束：`enrolledIn max 100`。
4. 保存为 `university-final.owl`，导出 Turtle `university.ttl`。

### Step 2 Fuseki 部署与查询脚本准备
1. Fuseki 建 dataset `university` → 上传 `university.ttl`。
2. 写 3 条固定 SPARQL 存成 `.rq` 文件：
   - `query_students_by_course.rq`（查某课的学生）
   - `query_courses_by_teacher.rq`（查某老师教的课）
   - `query_inconsistent.rq`（查是否有矛盾实例）

### Step 3 Python 脚本串联（伪代码级步骤）
1. 电脑装 Python → `pip install requests rdflib owlready2`。
2. 新建 `kg_qa.py`：
   ```python
   # Step A: 接收用户自然语言问题
   question = input("请输入问题：")

   # Step B: 调用LLM API抽取意图（此处简化为硬编码映射）
   # 若问“Alice教什么课”，对应SPARQL文件 query_courses_by_teacher.rq

   # Step C: 向Fuseki发SPARQL请求
   import requests
   response = requests.post(
       "http://localhost:3030/university/query",
       data={"query": open("query_courses_by_teacher.rq").read()},
       headers={"Accept": "application/json"}
   )
   results = response.json()

   # Step D: 把JSON结果拼成文本喂给LLM生成回答
   context = str(results)
   prompt = f"基于以下知识库查询结果：{context}，用自然语言回答：Alice教什么课？"
   # 调用LLM生成最终回答...
   ```
3. 运行脚本，输入“Alice教什么课”，确认流程通了。

### Step 4 交付物清单

## 全程避坑复查（贯穿8周）

- **每周保存都用 Save As 带版本号**（week1-v1, week2-v1…），方便回滚。
- **推理机启动前先 Check consistency**，红报错再看 Explanation（右键红色类 → Explain），不要盲目关推理。
- **SPARQL 查不到数据先核对 Prefix**，你的 IRI 是 `example.org` 还是 `semanticweb.org`，必须和文件里一致。
- **对齐顶层本体时注意命名冲突**，导入 SUMO 后别把自己的类也命名成 `Agent`，可用 `lib:Person` 前缀隔离。
