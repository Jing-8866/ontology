# 本体建模8周完整保姆级实操手册（零基础→LLM结合）

## 预备工作：环境与概念准备
- **安装Java**：Protégé 5.x 需 **Java 11+**（推荐 Adoptium Temurin 17）。安装后命令行输入 `java -version`，预期反馈版本号。
- **下载Protégé**：访问 https://protege.stanford.edu/ ，下载 **Protégé Desktop**，解压后双击 `run.bat`(Win) 或 `Protege.app`(Mac)。
- **首次启动**：加载约30秒，出现主界面即成功。
- **核心概念速记**：
  - **Class（类）**：概念（如 Person）
  - **Property（属性）**：关系（如 teaches）
  - **Individual（个体）**：实例（如 Alice）

## 第1周：零基础入门——建类、属性、实例（University V1）
**目标**：用 Protégé 构建 `Person / Student / Teacher` 层级，产出 `university-v1.owl`。

### Step 1 新建本体
1. 菜单栏点击 **File → New**。
2. 切换至 **Active Ontology** 标签页。
3. 找到 **Ontology IRI** 输入框，删除默认值，输入：`http://www.example.org/university#`
4. 点击 **OK**。
5. **预期反馈**：左侧 Classes 面板显示 `owl:Thing` 根节点。

### Step 2 创建类（Classes）
1. 左侧点击 **Entities → Classes** 标签。
2. 选中 `owl:Thing`。
3. 点击工具栏 **Add subclass**（图标是带加号的方块）。
4. 弹窗输入 `Person`，回车。
5. 选中 `Person`，再次点击 **Add subclass**，输入 `Student`；同理建 `Teacher`。
6. **预期反馈**：层级树显示：
   ```
   owl:Thing
    └── Person
         ├── Student
         └── Teacher
   ```

### Step 3 创建对象属性（Object Properties）
1. 左侧切换至 **Object Properties** 标签。
2. 选中 `owl:topObjectProperty`。
3. 点击 **Add subproperty**，输入 `teaches`，回车。
4. 选中 `teaches`，右侧面板点 **Domain (intersection)** 旁的 **+**。
5. 弹窗选 `Teacher` → OK。
6. 点 **Range (union)** 旁的 **+**，选 `Student` → OK。
7. **预期反馈**：右侧显示 Domain: Teacher, Range: Student。

### Step 4 创建数据属性（Data Properties）
1. 左侧切换 **Data Properties**。
2. 选中 `owl:topDataProperty` → **Add subproperty** → 输入 `hasName`。
3. 右侧 **Range** → **+** → 选择 `xsd:string` → OK。
4. 同理建 `hasAge`，Range 选 `xsd:int`。

### Step 5 创建个体（Individuals）
1. 左侧切换 **Individuals by class**。
2. 展开 `Teacher` → 右键 `Teacher` → **Add individual**，输入 `Alice`。
3. 右侧 **Data property assertions** → **+** → 选 `hasName` → 值输 `"Alice"`（带引号）→ OK；再加 `hasAge` → `35`。
4. 展开 `Student` → 建 `Bob`，`hasName="Bob"`, `hasAge=20`。
5. 选中 `Alice`，右侧 **Object property assertions** → **+** → 选 `teaches` → 填 `Bob` → OK。
6. **预期反馈**：`Alice` 下出现 `teaches Bob` 的关系线。

### Step 6 保存
- **File → Save as** → 选 **RDF/XML**，文件名 `university-v1.owl` → 保存。

## 第2周：属性体系与约束（University V2）
**目标**：加 `Course` 类、`enrolledIn` 属性、基数约束。

1. **Classes** 下 `owl:Thing` 建 `Course`。
2. **Object Properties** 建 `enrolledIn`，Domain=`Student`，Range=`Course`。
3. 选中 `Student` 类，右侧 **SubClass Of** → **+** → **Class expression editor**。
4. 输入：`enrolledIn min 1 Course` → OK。（至少选1门）
5. 再 **+** → 输入：`enrolledIn max 5 Course` → OK。（最多5门）
6. 选中 `Student`，右侧 **Disjoint With** → **+** → 选 `Teacher`（互斥）。
7. **预期反馈**：`Student` 描述中出现 `enrolledIn min 1 Course` 等公理。
8. 保存为 `university-v2.owl`。

## 第3周：推理机初体验（自动分类与矛盾检测）
**目标**：用年龄约束让 HermiT 自动分类 Undergraduate/Graduate。

1. `Student` 下建子类 `UndergraduateStudent`、`GraduateStudent`。
2. 分别设二者 **Disjoint With** 对方。
3. 选中 `UndergraduateStudent`，**SubClass Of** → **+** → 输：
   `hasAge some xsd:int[<= 22]` → OK。
4. 选中 `GraduateStudent`，**SubClass Of** → **+** → 输：
   `hasAge some xsd:int[> 22]` → OK。
5. 建个体 `Charlie`，类型仅选 `Student`，`hasAge=25`。
6. 顶部菜单 **Reasoner → HermiT → Start reasoner**。
7. **预期反馈**：`Charlie` 自动出现在 `GraduateStudent` 下（虚线框）。
8. 给 `Charlie` 再加 `hasAge=20`，**Reasoner → Synchronize reasoner**。
9. **预期反馈**：红色报错 **Inconsistent Ontology**，解释窗口显示年龄矛盾。

## 第4周：本体工程方法论——领域本体与需求说明书
**目标**：按斯坦福七步法写需求书，建图书管理本体 `library-week4-v1.owl`。

1. 纸面/文档列出 **Competency Questions**：
   - “某书的作者是谁？”
   - “某作者写了哪些书？”
2. Protégé 新建本体 IRI `http://www.example.org/library#`。
3. **Classes** 建：`Book`, `Person`, `Author`(subClass of Person), `Category`, `Publisher`，并设两两 **Disjoint**。
4. **Object Properties**：`hasAuthor`(Book→Author), `belongsToCategory`(Book→Category), `publishedBy`(Book→Publisher)。
5. **Data Properties**：`hasTitle`(string), `hasISBN`(string), `hasYear`(int)。
6. 创建实例 `AI_Book_01`(Book)，挂 `hasTitle="人工智能导论"`，连 `Author` 实例 `ZhangSan`。
7. **预期反馈**：本体结构完整，无红报错。保存版本。

## 第5周：顶层本体对齐（SUMO / Schema.org）
**目标**：导入 SUMO，将自建类映射到顶层。

1. 下载 `sumo.owl`（IEEE SUO 官网）。
2. Protégé 菜单 **File → Import → Import Ontology...** → **From File** → 选 `sumo.owl` → Next → Finish。
3. 左侧 Classes 出现大量 `sumo:` 开头类。
4. 选中自建 `Person` → 右侧 **SubClass Of** → **+** → 输 `sumo:Human` 或从下拉选 → OK。
5. 同理 `Book` SubClass Of `sumo:Artifact`。
6. **Reasoner → Start HermiT**。
7. **预期反馈**：`ZhangSan` 实例在 `sumo:Human` 类下也能看到（继承推理）。
8. 保存。

## 第6周：SPARQL 查询与 Fuseki 三元组存储
**目标**：部署 Fuseki，用 SPARQL 查本体。

### Step 1 启动 Fuseki
1. 下载 Apache Jena Fuseki，解压运行 `fuseki-server.bat`。
2. 浏览器开 `http://localhost:3030`。

### Step 2 上传数据
1. **Manage Datasets → Add Dataset** → Name: `university` → Create。
2. **Upload data** → 选 Protégé 导出的 `university.ttl`（File → Export → Turtle）→ Upload。

### Step 3 执行 SPARQL
1. 切 **Query** 页，Dataset 选 `university`。
2. 输入：
   ```sparql
   PREFIX uni: <http://www.example.org/university#>
   PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
   SELECT ?s ?name WHERE {
     ?s rdf:type uni:Student .
     ?s uni:hasName ?name .
   }
   ```
3. 点 **Run Query**。
4. **预期反馈**：下方表格出现 `Bob` 等学生数据。

## 第7周：本体 + LLM Prompt 工程（减少幻觉）
**目标**：用 Manchester Syntax 片段约束大模型。

1. Protégé 选中 `Book` 类，右侧复制 **Description** 文本：
   `Class: Book SubClassOf: hasAuthor some Author`
2. 打开 LLM 对话框（如 ChatGPT），输入 Prompt：
   ```
   以下是图书本体的OWL定义：
   Class: Book SubClassOf: hasAuthor some Author
   Class: Author SubClassOf: Person
   问题：如果《深度学习》的作者是张三，张三能否是Book类型？请严格基于本体回答。
   ```
3. **预期反馈**：LLM 回答“不能，张三是Author/Person，与Book互斥”，而非编造。

## 第8周：综合实战 Demo——University KG + LLM Text-to-SPARQL
**目标**：Python 串联 Fuseki + LLM，实现自然语言问答。

### 项目结构
```
university_kg_demo/
├── kg_qa.py
├── sparql/
│   └── courses_by_teacher.rq
└── university.ttl
```

### `sparql/courses_by_teacher.rq`
```sparql
PREFIX uni: <http://www.example.org/university#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
SELECT ?course ?cName WHERE {
  ?t rdf:type uni:Teacher ; uni:hasName "Alice" ; uni:teaches ?s .
  ?s uni:enrolledIn ?course .
  ?course uni:hasName ?cName .
}
```

### `kg_qa.py`（核心完整代码）
```python
import requests, json

FUSEKI = "http://localhost:3030/university/query"
LLM_API = "https://api.openai.com/v1/chat/completions"
KEY = "sk-xxxx"

def run_sparql(q):
    r = requests.post(FUSEKI, data={"query": q}, headers={"Accept":"application/json"})
    return r.json()

def ask_llm(ctx, q):
    prompt = f"基于以下KG结果回答：{json.dumps(ctx,ensure_ascii=False)}\n问题：{q}\n仅基于结果回答。"
    r = requests.post(LLM_API, json={
        "model":"gpt-3.5-turbo","temperature":0,
        "messages":[{"role":"user","content":prompt}]
    }, headers={"Authorization":f"Bearer {KEY}"})
    return r.json()["choices"][0]["message"]["content"]

# SPARQL 生成 Prompt（LLM Text-to-SPARQL 简化版规则映射）
def main():
    q = input("问题> ")
    sparql = open("sparql/courses_by_teacher.rq").read()
    kg = run_sparql(sparql)
    ans = ask_llm(kg, q)
    print("回答:", ans)

if __name__ == "__main__":
    main()
```

### 运行步骤
1. 启动 Fuseki 并确保 `university` dataset 已上传 ttl。
2. 终端 `pip install requests` → `python kg_qa.py`。
3. 输入：`Alice教了哪些课？`
4. **预期反馈**：程序打印 SPARQL 查询结果，LLM 生成自然语言回答（如“人工智能导论”）。

## 附：新手常见错误对照表（贯穿8周）
- **实例建成类**：`Bob` 应是 Individual，不是 Class。→ 导致无法推理。
- **忘设 Domain/Range**：`teaches` 谁都能用。→ 语义污染。
- **类名带空格**：`Computer Science` ❌ → `ComputerScience` ✅。
- **期望LLM猜常识**：推理机只做逻辑推导，不猜“老师通常比学生大”。
- **OWL与DB混淆**：OWL 是开放世界（没说不存在≠不存在），DB是封闭世界。
